# 데이터 연계 솔루션 구축

## Overview

API 기반 이기종 데이터베이스간 연계 플랫폼을 구축한 프로젝트입니다.  
여러 서비스의 데이터베이스를 연계하고 관리하는 구조에서 운영 안정성과 확장성을 높이기 위해 MSA 기반 구조, 서비스 이중화, CI/CD, 모니터링 환경을 함께 구성했습니다.

- **기간:** 2026.02 ~ 2026.06
- **참여 인원:** 2명
- **역할:** 백엔드 개발, 서비스 구조 설계, 배포/모니터링 환경 구성, SAP 연계 프로세스 구현
- **사용기술:** Java, Spring Boot, Spring Cloud, Vue.js, MS-SQL, Nginx, Jenkins, CI/CD, Prometheus, Grafana, Ubuntu, SAP JCo Connector, RFC, SAP HANA DB

## Problem

- 단일 서비스 장애가 주요 서비스 중단으로 이어질 수 있는 구조
- 다수 인터페이스 처리 과정에서 API 병목 현상 발생
- 서비스 이중화 과정에서 스케줄러 중복 실행 발생
- 수동 배포와 체계화되지 않은 빌드 과정으로 인해 배포 이력 관리와 운영 대응 어려움
- 서비스 상태, 메모리 사용량, 리소스 이상 징후 확인, 로그 관리 체계 부족

## Solution

### 1. MSA 기반 구조와 서비스 이중화

모놀리식 아키텍처에서 서비스 경계를 나누고, Gateway를 통해 요청 흐름을 관리하는 MSA 기반 구조를 적용
서비스 이중화를 통해 단일 장애 지점의 영향을 줄이고, 서비스별 배포와 확장이 가능하도록 구성

### 2. API 병목 개선

다수 인터페이스 처리 과정에서 MSSQL DLL에서 발생하던 API 병목 현상을 해결

### 3. Quartz Clustering 적용

Quartz Clustering을 적용해 이중화 환경에서 스케줄러 실행 상태를 공유하고, 동일 배치가 중복 실행되지 않도록 구성

### 4. Jenkins 기반 CI/CD 구성

빌드와 배포 과정이 체계적으로 관리되지 않는 문제를 개선하기 위해 Jenkins 기반 CI/CD 파이프라인을 구성

### 5. Prometheus, Grafana 모니터링 구성

서비스 운영 중 현재 상태와 과거 이력 확인이 어려워 해당 문제를 개선하기 위해 Prometheus와 Grafana 기반 모니터링 환경을 구성
메모리 사용량, 서버 상태, 주요 리소스 지표를 확인할 수 있도록 구성해 메모리 누수와 리소스 이상 징후를 추적 가능

## Result

- 단일 장애 지점 발생이 줄어들었고 에러 발생 시 추적이 편함
- 서비스별 배포와 확장이 가능한 구조로 무중단으로 서비스 별 배포가 가능하여 운영 편의성 상승
- 50만 건 처리 시간 4분 30초 -> 1분 30초 단축
- Quartz Clustering으로 스케줄러 중복 실행 문제 해결
- Jenkins 기반 CI/CD 구성으로 배포 자동화 및 운영 효율 향상
- Prometheus, Grafana 기반 모니터링으로 서비스 상태 가시성 확보, 서비스 별 로그를 대시보드로 확인 가능
- ETL 및 멀티스레드 비동기 처리 성능을 개선하며 CPU, 메모리 사용량, 실행 중인 스레드 수를 모니터링해 병목 지점 분석
