# 사용자 맞춤형 온수난방 시스템 개발

## Overview

기존 온수·난방 제어 시스템과 연동되는 Spring 기반 웹 서비스와 관리자 대시보드를 개발한 프로젝트입니다.  
IoT 기기 통신, FOTA, 실시간 이벤트 전달, 데이터 처리 파이프라인을 구현했습니다.

- **기간:** 2023.10 ~ 2026.01
- **개발 기간:** 2023.10 ~ 2024.05
- **참여 인원:** 2명
- **역할:** 백엔드 API 개발, IoT 통신 기능 구현, 실시간 데이터 처리, 관리자 대시보드 개발
- **사용기술:** Java, Spring Boot, Vue.js, MySQL, Kafka, MQTT, Firebase, FOTA, R, Ubuntu, CentOS

## My Role

- Spring Boot 기반 웹 서비스 개발
- MQTT 기반 기기 통신 기능 구현
- FOTA 기능 구현
- Kafka 기반 실시간 데이터 처리 파이프라인 구축
- Firebase 기반 앱 알림 기능 구현
- Vue.js 관리자 대시보드 개발
- R 기반 사용자 사용 패턴 분석 및 데이터 가공 로직 개발

## Problem

기기 펌웨어를 직접 전송하는 구조에서 전송 지연, 기기 오작동, 다운로드 실패 문제가 발생했습니다.  
또한 IoT 기기에서 발생하는 데이터와 이벤트를 안정적으로 수집하고, 운영자가 상태를 확인할 수 있는 구조가 필요했습니다.

## Solution

### 1. 서버 중계 다운로드 방식 전환

대용량 펌웨어를 직접 전송하지 않고 서버 중계 다운로드 방식으로 전환해 배포 안정성을 높였습니다.

### 2. MQTT 기반 기기 통신

MQTT를 활용해 기기와 서버 간 통신 구조를 구현했습니다.  
기기 상태와 이벤트를 안정적으로 전달할 수 있도록 구성했습니다.

### 3. Kafka 기반 데이터 처리

Kafka 기반 실시간 데이터 처리 파이프라인을 구축해 IoT 데이터를 안정적으로 스트리밍 처리했습니다.

### 4. 관리자 대시보드와 알림

Vue.js 관리자 대시보드를 개발하고 Firebase 기반 앱 알림 기능을 구현해 실시간 이벤트 전달과 모니터링 기능을 제공했습니다.

## Result

- 펌웨어 배포 안정성 확보
- MQTT 기반 기기 통신 및 FOTA 기능 구현
- Kafka 기반 IoT 데이터 스트리밍 처리 구조 구축
- Firebase 기반 실시간 이벤트 알림 제공
- Vue.js 관리자 대시보드를 통한 모니터링 및 제어 기능 제공

## What I Learned

IoT 서비스에서는 기능 구현뿐 아니라 기기 상태, 네트워크 상황, 펌웨어 배포 실패 가능성까지 함께 고려해야 한다는 점을 배웠습니다.  
또한 실시간 이벤트 처리는 메시지 흐름과 운영 가시성이 함께 설계되어야 안정적으로 운영될 수 있음을 경험했습니다.
