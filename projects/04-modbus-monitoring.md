# 캐스케이드 제어·모니터링 시스템 고도화

## Overview

기기와 시스템 간 통신 불안정 문제를 개선하고, 원격 모니터링과 네트워크 제어 환경을 구축한 프로젝트입니다.

- **기간:** 2024.07 ~ 2025.04
- **참여 인원:** 1명
- **역할:** 통신 구조 개선, 데스크톱 애플리케이션 개발, 예외 처리 및 로그 시스템 구축
- **사용기술:** C#, .NET Framework, WinForm, MSSQL, Modbus TCP/IP, Serial Communication

## My Role

- C#/.NET Framework 기반 WinForm 애플리케이션 개발
- Modbus TCP/IP 기반 통신 기능 구현
- 네트워크 상태 체크 로직 기반 서버/로컬 통합 구조 구현
- 통신 구간별 예외 처리 및 로그 시스템 구축
- 원격 모니터링 및 네트워크 제어 환경 구성

## Problem

기기와 시스템 간 통신 노이즈로 제어 실패가 발생했고, 서버/로컬 프로그램이 이원화되어 운영과 장애 대응이 복잡했습니다.  
장애가 발생했을 때 어느 통신 구간에서 문제가 발생했는지 파악하기 어려운 점도 문제였습니다.

## Solution

### 1. Modbus TCP/IP 기반 통신 구조 전환

불안정한 통신 구조를 Modbus TCP/IP 기반으로 전환해 요청/응답 흐름을 명확히 했습니다.

### 2. 단일 애플리케이션 통합

서버/로컬로 나뉘어 있던 프로그램을 네트워크 상태 체크 로직 기반 단일 애플리케이션으로 통합했습니다.

### 3. 예외 처리와 로그 시스템

통신 구간별 예외 처리와 로그 시스템을 구축해 장애 발생 시 원인을 더 빠르게 파악할 수 있도록 개선했습니다.

## Result

- 통신 실패율 감소 및 데이터 신뢰성 향상
- 서버/로컬 이원화 프로그램을 단일 애플리케이션으로 통합
- 통신 구간별 로그를 통해 장애 분석 시간 단축
- 원격 모니터링 및 네트워크 제어 환경 구축
- 온수/난방 개별 Serial 통신을 단일 채널 처리 구조로 통합

## What I Learned

장비 통신 시스템에서는 통신 성공 여부뿐 아니라 실패 원인을 추적할 수 있는 로그와 예외 처리가 중요하다는 점을 배웠습니다.  
또한 운영자가 실제로 장애를 분석하고 대응할 수 있는 구조를 만드는 것이 기능 구현만큼 중요하다는 점을 경험했습니다.
