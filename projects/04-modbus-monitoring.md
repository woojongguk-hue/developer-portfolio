# 캐스케이드 제어·모니터링 시스템 고도화

## Overview

캐스케이드 보일러와 시스템 간 통신 불안정 문제를 개선하고, 원격 모니터링과 네트워크 제어 환경을 구축한 프로젝트입니다.
기존 온수/난방을 개별로 Serial 통신을 하던 구조를 단일 채널 처리 구조로 통합하였습니다.

- **기간:** 2024.07 ~ 2025.04
- **참여 인원:** 1명
- **역할:** 통신 구조 개선, 데스크톱 애플리케이션 개발, 예외 처리 및 로그 시스템 구축
- **사용기술:** C#, .NET Framework, WinForm, MSSQL, Modbus(RS485,TCP/IP)

## Problem

- 기기와 시스템 간 통신 노이즈로 제어 실패가 빈번히 발생하였고, 서버/로컬 프로그램이 이원화되어 운영과 장애 대응이 복잡
- 장애가 발생했을 때 어느 통신 구간에서 문제가 발생했는지 파악이 어려움

## Solution

### 1. Modbus TCP/IP 기반 통신 구조 전환

기존 불안정한 일반 통신 구조를 Modbus 프로토콜 기반으로 전환해 요청/응답 흐름을 명확히 하여 통신 안정성 확보

### 2. 단일 애플리케이션 통합

서버/로컬로 나뉘어 있던 프로그램을 네트워크 상태 체크 로직 기반 단일 애플리케이션으로 통합

### 3. 예외 처리와 로그 시스템

통신 구간별 예외 처리와 로그 시스템을 구축해 장애 발생 시 원인을 더 빠르게 파악할 수 있도록 개선

## Result

- 통신 실패율 감소 및 데이터 신뢰성 향상
- 서버/로컬 이원화 프로그램을 단일 애플리케이션으로 통합
- 통신 구간별 로그를 통해 장애 분석 시간 단축
- 원격 모니터링 및 네트워크 제어 환경 구축
